# Guide de contribution

Merci de votre intérêt pour contribuer à StarPanel !

## Comment contribuer

### Signaler un bug

1. Vérifier que le bug n'a pas déjà été signalé dans les [Issues](https://github.com/SAA-Gaming/issues)
2. Créer une nouvelle issue avec :
   - Description claire du problème
   - Étapes pour reproduire
   - Comportement attendu vs comportement actuel
   - Version de l'app et du host
   - Logs si disponibles

### Proposer une fonctionnalité

1. Créer une issue avec le label "enhancement"
2. Décrire la fonctionnalité et son utilité
3. Attendre la discussion avant d'implémenter

### Contribuer du code

1. Fork le repository
2. Créer une branche pour votre fonctionnalité (`git checkout -b feature/ma-fonctionnalite`)
3. Commiter vos changements (`git commit -m 'Ajout de ma fonctionnalité'`)
4. Push vers la branche (`git push origin feature/ma-fonctionnalite`)
5. Ouvrir une Pull Request

### Standards de code

- **Android** : Suivre les conventions Kotlin et les guidelines Android
- **C#** : Suivre les conventions C# et les guidelines .NET
- **Commits** : Messages clairs en français ou anglais
- **Tests** : Ajouter des tests pour les nouvelles fonctionnalités

### Profils communautaires

Les profils partagés doivent :
- Être testés et fonctionnels
- Inclure une description claire
- Suivre le format JSON schema
- Être documentés (touches mappées, etc.)

## Questions ?

N'hésitez pas à ouvrir une issue pour poser des questions !
